A "flank" is a collection (set) of users who responded to some number (set) of questions in the same way. A "counterflank" is the converse of a flank—i.e., a set of users with exactly the opposite set of responses to the corresponding flank.

If users across the "flank"‐"counterflank" disjoint 2-tuple have answered other questions in the same way, these answers are considered "revelations" because users in both groups agree. The larger the "flank"‐"counterflank" 2-tuple in terms of both number of users and number of questions, the more significant the revelations become.

Due to long processing times, flanks, counterflanks and revelations are not processed in realtime. We are working to improve these processing times so that users can received realtime feedback to their answers. In the meantime, contact support@newsflanks.com if you would like to see where your responses fit into the larger data sets. We can also tell you what revelations you contributed to, and how significant those revelations (truths) appear to be.
